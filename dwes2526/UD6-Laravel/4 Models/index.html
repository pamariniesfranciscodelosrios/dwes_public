Models. Eloquent ORM

Creating Models in Laravel

These are the core concepts you need to understand when building models in Laravel using Eloquent ORM.

Documentation link

Full Documentation Link for Eloquent ORM

üîπ 1. Model Basics

  • Every Eloquent model class represents a single table in your database.
  • It should extend the base Illuminate\Database\Eloquent\Model class.

Example:

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Post extends Model
{
    //
}

By default:

  • The table associated with the model is the plural snake_case version of the class name (Post ‚Üí posts).
  • The primary key is id.
  • Timestamps (created_at, updated_at) are managed automatically.

Muy importante

Fijarse en el nombre de losmodelos (singular may√∫scula) y de las tablas (plural min√∫cula) (Post ‚Üí posts). Y seguir la convenci√≥n para que no haya que realizar ajustes adicionales en el Framework

If needed, you can override the default table or primary key:

protected $table = 'blog_posts';
protected $primaryKey = 'post_id';

üîπ 2. Mass Assignment Protection

Eloquent protects against mass assignment vulnerabilities.

  • Use $fillable to define which attributes can be mass-assigned :
protected $fillable = ['title', 'body', 'published'];
  • Or, use $guarded to define which attributes cannot be mass-assigned :
protected $guarded = ['id'];

Only one of these should typically be used per model.

üîπ 3. Relationships Between Models

Laravel makes it simple to define relationships between models. Here are the most common:

  • hasOne: One-to-one
  • hasMany: One-to-many
  • belongsTo: Inverse of hasOne or hasMany
  • belongsToMany: Many-to-many
  • hasManyThrough: Distant one-to-many

Example:

In a blog app, one Post has many Comments:

// In Post.php
public function comments()
{
    return $this->hasMany(Comment::class);
}

And each comment belongs to a post:

// In Comment.php
public function post()
{
    return $this->belongsTo(Post::class);
}

This allows you to use:

$post->comments;       // Get all comments for a post
$comment->post;        // Get the post of a comment

üîπ 4. Eager Loading

To avoid the N+1 query problem when retrieving relationships, use eager loading :

$posts = Post::with('comments')->get();

This loads all posts and their comments in just 2 queries instead of one per post.

üîπ 5. CRUD Operations

Eloquent offers elegant methods for creating, reading, updating, and deleting records.

// Create
$post = Post::create([
    'title' => 'My Title',
    'body' => 'My content'
]);

// Read
$post = Post::find(1);      // By ID
$posts = Post::all();       // All posts

// Update
$post->update(['title' => 'Updated Title']);

// Delete
$post->delete();

You can also use query builder chaining:

Post::where('published', true)->orderBy('created_at', 'desc')->get();

üîπ 6. Query Scopes

Scopes allow you to define reusable conditions in your model.

Example:

public function scopePublished($query)
{
    return $query->where('published', true);
}

Then use it like this:

$posts = Post::published()->get();

You can also pass parameters:

public function scopeOfType($query, $type)
{
    return $query->where('type', $type);
}

üîπ 7. Timestamps

Eloquent automatically maintains created_at and updated_at fields.

To disable this behavior:

public $timestamps = false;

If you use custom timestamp column names:

const CREATED_AT = 'creation_date';
const UPDATED_AT = 'update_date';

üîπ 8. Attribute Casting

Laravel allows you to automatically cast attributes to common data types:

protected $casts = [
    'is_published' => 'boolean',
    'published_at' => 'datetime',
    'options' => 'array',
];

This means $model->is_published will return a true or false boolean, for example.

üîπ 9. Accessors and Mutators

Customize how values are retrieved (accessors) or set (mutators):

Accessor example:

public function getTitleAttribute($value)
{
    return ucfirst($value); // Capitalizes the first letter
}

Mutator example:

public function setTitleAttribute($value)
{
    $this->attributes['title'] = strtolower($value);
}

üîπ 10. Soft Deletes (Optional)

If you want to "delete" records without removing them from the database:

php artisan make:migration add_deleted_at_to_posts_table --table=posts

Then add a deleted_at column:

$table->softDeletes();

In the model:

use Illuminate\Database\Eloquent\SoftDeletes;

class Post extends Model
{
    use SoftDeletes;
}

Now calling $post->delete() won't remove the row, just mark it as deleted.

‚úèÔ∏è STEP 3: Define the Migration

Edit the migration file:

üìÑ database/migrations/xxxx_xx_xx_create_posts_table.php

public function up(): void
{
    Schema::create('posts', function (Blueprint $table) {
        $table->id();
        $table->string('title');
        $table->text('body');
        $table->boolean('published')->default(true);
        $table->timestamps();
    });
}

Then run the migration:

‚úèÔ∏èSTEP 4: Define the Model Logic

Edit:

üìÑ app/Models/Post.php

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Factories\HasFactory;

class Post extends Model
{
    use HasFactory;

    protected $fillable = ['title', 'body', 'published'];

    // Accessor
    public function getTitleAttribute($value)
    {
        return ucfirst($value); // Capitalize the first letter of the title
    }

    // Scope
    public function scopePublished($query)
    {
        return $query->where('published', true); // Filter to get only published posts
    }

    // Auto-insert sample posts when the model is first loaded
    protected static function booted()
    {
        if (self::count() === 0) { // Check if there are no posts
            // Insert sample posts
            self::create([
                'title' => 'welcome post',
                'body' => 'This is a welcome post created automatically.',
                'published' => true,
            ]);

            self::create([
                'title' => 'draft example',
                'body' => 'This post is a draft and won‚Äôt show on the page.',
                'published' => false,
            ]);
        }
    }
}

            <p>{{ $post->body }}</p>
            <hr>
        </div>
    @empty
        <p>No posts found.</p>
    @endforelse
</body>
</html>
