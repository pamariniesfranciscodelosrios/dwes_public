ðŸ’» Laravel Migrations Activity:

Step 1: create the project

Requirements:

folder Sesion3
composer create-project laravel/laravel learn-migrations
cd learn-migrations
php artisan serve

Step 2: Set Up the Database

  1. Open .env
  2. Update DB configuration: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=learn_migrations
DB_USERNAME=root
DB_PASSWORD=
  3. Create the database in MySQL: 
    CREATE DATABASE learn_migrations;

Step 3: Understand Migrations Using the Laravel Docs

Visit: https://laravel.com/docs/12.x/migrations

Read these key sections:

  • Introduction
  • Creating Migrations
  • Running Migrations
  • Rolling Back
  • Modifying Columns
  • Foreign Keys

Step 4: Create and Run Migrations

  1. Generate a migration: 
    php artisan make:migration create_students_table
  2. Define the schema in database/migrations/xxxx_xx_xx_create_students_table.php: 
    public function up(): void
{
    Schema::create('students', function (Blueprint $table) {
        $table->id();
        $table->string('name');
        $table->string('email')->unique();
        $table->timestamps();
    });
}
  3. Run migration: 
    php artisan migrate
  4. Verify in database: the students table should now exist.

Step 5: Modify Migrations

  1. Create another migration to add a column : 
    php artisan make:migration add_age_to_students_table
  2. Edit the new migration: 
    public function up(): void
{
    Schema::table('students', function (Blueprint $table) {
        $table->integer('age')->nullable();
    });
}
  3. Run it: 
    php artisan migrate

Step 6: Rollback and Refresh

  • Rollback last batch: 
    php artisan migrate:rollback
  • Rollback all and rerun: 
    php artisan migrate:refresh
  • Fresh migration: 
    php artisan migrate:fresh

Personal Activity: Build a Simple Shop Database

Objective

Build a Laravel project called simple-shop focused on learning how to use migrations by designing a database table for products in a shop .

Instructions

1. Set Up a New Laravel Project

folder: sesion3 
composer create-project laravel/laravel simple-shop
cd simple-shop
php artisan serve

2. Configure Your Database

  1. Open the .env file and update the database settings: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=simple_shop
DB_USERNAME=root
DB_PASSWORD=
  2. Create the database in MySQL (Optional): 
    CREATE DATABASE simple_shop;

3. Create a Migration for products Table

Run the following command:

php artisan make:migration create_products_table

4. Add 10 Columns to the Products Table

Edit the generated migration file in database/migrations/ to include the following fields:

public function up(): void
{
    Schema::create('products', function (Blueprint $table) {
        $table->id();
        $table->string('name');
        $table->text('description')->nullable();
        $table->decimal('price', 8, 2);
        $table->integer('stock');
        $table->string('sku')->unique();
        $table->boolean('is_active')->default(true);
        $table->unsignedBigInteger('category_id')->nullable();
        $table->string('brand')->nullable();
        $table->timestamp('published_at')->nullable();
        $table->timestamps();
    });
}

ðŸ”¢ This gives you 10 columns :

  • id (auto)
  • name
  • description
  • price
  • stock
  • sku
  • is_active
  • category_id
  • brand
  • published_at

5. Run the Migration

php artisan migrate

Verify that the products table was created in your database.

6. Add a New Column with a New Migration

Add a new column weight:

php artisan make:migration add_weight_to_products_table

Then edit it:

public function up(): void
{
    Schema::table('products', function (Blueprint $table) {
        $table->decimal('weight', 6, 2)->nullable()->after('price');
    });
}

Run it:

php artisan migrate

7. Try Rollback and Refresh

  • Roll back last migration: 
    php artisan migrate:rollback
  • Roll back and rerun all: 
    php artisan migrate:refresh
  • Start fresh: 
    php artisan migrate:fresh
