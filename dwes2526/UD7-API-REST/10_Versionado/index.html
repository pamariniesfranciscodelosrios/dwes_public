UD7. S10 versiones

1) Versionado por URL (recomendado)

Estructura de rutas

En routes/api.php:

use Illuminate\Support\Facades\Route;

Route::prefix('v1')->group(function () {
    // endpoints v1
    Route::get('/tasks', [\App\Http\Controllers\Api\V1\TaskController::class, 'index']);
});

Route::prefix('v2')->group(function () {
    // endpoints v2
    Route::get('/tasks', [\App\Http\Controllers\Api\V2\TaskController::class, 'index']);
});

Estructura de controladores (carpetas/namespace)

Crea controladores separados:

  • app/Http/Controllers/Api/V1/TaskController.php
  • app/Http/Controllers/Api/V2/TaskController.php

Ventaja: clarísimo y fácil de mantener. Coste: duplicas algo de código si no abstraes.

Recomendación de diseño

  • Mantén el modelo igual (si no cambia la BD).
  • Versiona lo que cambia: Request , Resource , Controller , Policies .

Ejemplo típico:

  • V1 devuelve: {id, title}
  • V2 devuelve: {id, title, status, links}

2) Versionado por cabecera (Accept / custom header)

En vez de /v1, el cliente manda una cabecera, por ejemplo:

  • Accept: application/vnd.tuapp.v1+json o
  • X-API-Version: 1

Laravel no trae esto “hecho” por defecto: lo normal es añadir un middleware que lea la cabecera y redirija a rutas o setee un “contexto” de versión.

Ejemplo simple de idea (middleware):

  • lee X-API-Version
  • si es 2, enruta a grupo v2 o aplica bindings/transformers v2

Ventaja: URLs limpias. Inconveniente: más complejo de implementar y de depurar.

3) Versionado por subdominio

Ejemplos:

  • https://api-v1.midominio.com/...
  • https://v2.api.midominio.com/...

Laravel permite agrupar rutas por dominio:

Route::domain('v1.api.midominio.com')->group(function () {
    // rutas v1
});

Ventaja: separación fuerte. Inconveniente: DNS/SSL y despliegue más pesado.

¿Cómo se hace?

A) Qué versionar

Versiona cuando cambie el “contrato”:

  • Campos JSON (añadir/quitar/renombrar)
  • Validaciones y reglas
  • Semántica de endpoints (paginación, filtros, status codes)
  • Formato de errores

No hace falta versionar si solo cambias implementación interna sin afectar al cliente.

B) Cómo evitar duplicación (muy recomendado)

Comparte la lógica en una capa común y deja el “contrato” por versión.

Estructura habitual:

  • app/Services/TaskService.php (lógica común)
  • Api/V1/TaskController y Api/V2/TaskController (contrato)
  • Api/V1/TaskResource y Api/V2/TaskResource
  • Api/V1/Requests/... y Api/V2/Requests/...

Así no duplicas queries y reglas de negocio.

C) Política de cambios (para alumnado)

  • v1 : estable
  • v2 : cambios “rompedores”
  • Mantienes v1 durante X tiempo (en clase: hasta fin de práctica)
  • Documentas diferencias en README

Ejemplo típico de cambio entre v1 y v2

  • v1 GET /api/v1/tasks devuelve:
{ "data": [{ "id": 1, "title": "..." }] }
  • v2 GET /api/v2/tasks devuelve:
{ "data": [{ "id": 1, "title": "...", "status": "open" }], "meta": {...} }

laravel new tickets-api

Crea el archivo de BD:

# Linux/Mac

New-Item -ItemType File -Path database/database.sqlite

DB_CONNECTION=sqlite

2.3 Factory

<?php

namespace Database\Factories;

use App\Models\Ticket;
use Illuminate\Database\Eloquent\Factories\Factory;

/**
 * @extends Factory<Ticket>
 */
class TicketFactory extends Factory
{
    protected $model = Ticket::class;

    public function definition(): array
    {
        $statuses = ['open', 'in_progress', 'closed'];

        return [
            'title' => $this->faker->sentence(4),
            'description' => $this->faker->optional(0.7)->paragraph(),
            'status' => $this->faker->randomElement($statuses),
            'priority' => $this->faker->numberBetween(1, 5),
        ];
    }
}

4) Resources (contratos distintos)

4.1 V1 Resource (simple)

app/Http/Resources/V1/TicketResource.php

<?php

namespace App\Http\Resources\V1;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

/** @mixin \App\Models\Ticket */
class TicketResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'title' => $this->title,
            'status' => $this->status,
        ];
    }
}

4.2 V2 Resource (extendido + links.self)

app/Http/Resources/V2/TicketResource.php

<?php

namespace App\Http\Resources\V2;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

/** @mixin \App\Models\Ticket */
class TicketResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'title' => $this->title,
            'description' => $this->description,
            'status' => $this->status,
            'priority' => $this->priority,
            'created_at' => optional($this->created_at)->toISOString(),
            'links' => [
                'self' => "/api/v2/tickets/{$this->id}",
            ],
        ];
    }
}

5) Controladores por versión

5.1 V1 Controller (CRUD simple)

app/Http/Controllers/Api/V1/TicketController.php

<?php

namespace App\Http\Controllers\Api\V1;

use App\Http\Controllers\Controller;
use App\Http\Requests\V1\StoreTicketRequest;
use App\Http\Requests\V1\UpdateTicketRequest;
use App\Http\Resources\V1\TicketResource;
use App\Models\Ticket;
use Illuminate\Http\Response;

class TicketController extends Controller
{
    public function index()
    {
        $tickets = Ticket::query()->orderByDesc('id')->get();
        return TicketResource::collection($tickets);
    }

    public function show(Ticket $ticket)
    {
        return new TicketResource($ticket);
    }

    public function store(StoreTicketRequest $request)
    {
        $data = $request->validated();

        // defaults si no vienen
        $data['status'] = $data['status'] ?? 'open';
        $data['priority'] = $data['priority'] ?? 3;

        $ticket = Ticket::create($data);

        return (new TicketResource($ticket))
            ->response()
            ->setStatusCode(Response::HTTP_CREATED);
    }

    public function update(UpdateTicketRequest $request, Ticket $ticket)
    {
        $ticket->update($request->validated());
        return new TicketResource($ticket);
    }

    public function destroy(Ticket $ticket)
    {
        $ticket->delete();
        return response()->noContent(); // 204
    }
}

5.2 V2 Controller (CRUD + index con filtros/búsqueda/paginación)

app/Http/Controllers/Api/V2/TicketController.php

<?php

namespace App\Http\Controllers\Api\V2;

use App\Http\Controllers\Controller;
use App\Http\Requests\V2\StoreTicketRequest;
use App\Http\Requests\V2\UpdateTicketRequest;
use App\Http\Resources\V2\TicketResource;
use App\Models\Ticket;
use Illuminate\Http\Request;
use Illuminate\Http\Response;

class TicketController extends Controller
{
    public function index(Request $request)
    {
        $query = Ticket::query()->orderByDesc('id');

        if ($request->filled('search')) {
            $search = $request->string('search');
            $query->where('title', 'like', '%' . $search . '%');
        }

        if ($request->filled('status')) {
            $status = $request->string('status');
            $query->where('status', $status);
        }

        $perPage = (int) $request->input('per_page', 10);
        if ($perPage < 1) $perPage = 1;
        if ($perPage > 50) $perPage = 50;

        $paginator = $query->paginate($perPage)->appends($request->query());

        return TicketResource::collection($paginator);
    }

    public function show(Ticket $ticket)
    {
        return new TicketResource($ticket);
    }

    public function store(StoreTicketRequest $request)
    {
        $data = $request->validated();
        $data['status'] = $data['status'] ?? 'open';
        $data['priority'] = $data['priority'] ?? 3;

        $ticket = Ticket::create($data);

        return (new TicketResource($ticket))
            ->response()
            ->setStatusCode(Response::HTTP_CREATED);
    }

    public function update(UpdateTicketRequest $request, Ticket $ticket)
    {
        $ticket->update($request->validated());
        return new TicketResource($ticket);
    }

    public function destroy(Ticket $ticket)
    {
        $ticket->delete();
        return response()->noContent();
    }
}

7) Tests Feature (4 obligatorios)

tests/Feature/ApiVersioningTest.php

<?php

namespace Tests\Feature;

use App\Models\Ticket;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class ApiVersioningTest extends TestCase
{
    use RefreshDatabase;

    public function test_v1_index_returns_simple_contract(): void
    {
        Ticket::factory()->count(3)->create();

        $res = $this->getJson('/api/v1/tickets');

        $res->assertOk()
            ->assertJsonStructure([
                'data' => [
                    ['id', 'title', 'status']
                ]
            ]);

        // Asegura que NO expone campos de v2
        $this->assertArrayNotHasKey('links', $res->json('data.0'));
        $this->assertArrayNotHasKey('priority', $res->json('data.0'));
    }

    public function test_v2_show_returns_extended_contract_with_self_link(): void
    {
        $ticket = Ticket::factory()->create([
            'status' => 'open',
            'priority' => 3,
        ]);

        $res = $this->getJson("/api/v2/tickets/{$ticket->id}");

        $res->assertOk()
            ->assertJsonStructure([
                'data' => [
                    'id', 'title', 'description', 'status', 'priority', 'created_at',
                    'links' => ['self'],
                ]
            ])
            ->assertJsonPath('data.links.self', "/api/v2/tickets/{$ticket->id}");
    }

    public function test_v2_index_filters_by_status(): void
    {
        Ticket::factory()->count(2)->create(['status' => 'open']);
        Ticket::factory()->count(3)->create(['status' => 'closed']);

        $res = $this->getJson('/api/v2/tickets?status=closed');

        $res->assertOk();

        $items = $res->json('data');
        $this->assertNotEmpty($items);

        foreach ($items as $item) {
            $this->assertSame('closed', $item['status']);
        }
    }

    public function test_v2_index_paginates_and_respects_per_page(): void
    {
        Ticket::factory()->count(25)->create();

        $res = $this->getJson('/api/v2/tickets?per_page=7&page=2');

        $res->assertOk()
            ->assertJsonStructure([
                'data',
                'links',
                'meta',
            ]);

        $this->assertCount(7, $res->json('data'));
        $this->assertSame(7, $res->json('meta.per_page'));
        $this->assertSame(2, $res->json('meta.current_page'));
    }
}

# Tickets API (Laravel) — Versionado por URL

## Requisitos
- PHP 8.2+

## Instalación
```bash
composer install

  • PUT /api/v2/tickets/{ticket}
  • DELETE /api/v2/tickets/{ticket}
