Testing en Laravel

Parte fundamental dentro de cada proyecto profesional. Se pueden automatizar las pruebas de nuestro sistema

Tipos de Test en Laravel

  1. Feature test de características para probar el circuito completo de la aplicación
  2. Unit Para probar cada elemento, cada parte del programa, por ejemplo el funcionamiento de un controlador concreto

Ambos test tienen la misma sintaxis y el mismo formato de trabajo

CREACIÓN DE TEST

  1. Crear Proyecto 
    composer create-project laravel/laravel testexample

INFO Application key set successfully.

  1. Configurar base de datos
    En lugar de conectar a mysql como hace él, lo haremos con SQLITE:
    fichero: .env
//Cambiamos la conexión mysql por
DB_CONNECTION=sqlite

  1. Testing PHPUnit 

    Cambiamos la conexión mysql por
DB_CONNECTION=sqlite
    Hay dos tecnologías, en MVC va a usar PHPUNIT y PEST Fin mismo pero sintaxis diferente, más parecida al FRONT.

  2. Creación test Por defecto make:test crea Feature Test, test de características para probar el circuito completo de la aplicación 

    //convención camel case
php artisan make:test UserTest
    INFO Test [tests/Feature/UserTest.php] created successfully.

Creamos también el unitario 

//convención camel case
php artisan make:test UserTest --unit

INFO Test [tests/Unit/UserTest.php] created successfully.

APLICANDO TDD

https://youtu.be/3xQt_tAj1e0?t=2646

Importante destacar que estis test heredan de la clase TestCase

  1. Ejecución de test Primero deben fallar 
    php artisan test
    Tests: 2 failed, 1 passed (3 assertions)

Si subes arriba en la consola , con la respuesta puedes verlo mejor, FAIL Tests\Feature\UserTest ⨯ get user list 0.08s
⨯ get detail ✓ get non existing user detail

también podríamos llevarnos esa respuesta a un archivo de log por ejemplo.

  1. Depurando test Vamos a entender porqué ha fallado el primer test, Si miras en la respuesta lo puedes ver: Expected response status code [200] but received 404. Failed asserting that 404 is identical to 200.

Esperaba un 200 y ha obtenido un 404 ya que la función no está implementada

Lo mismo con el otro fallo. - El tercero si que funciona

Siguiendo filosofia TDD; solucionamos error por error

  1. Solucionamos el error 1 Get User List. Ahora acudimos a las rutas

Lo primero en la ruta api.php escribimos la ruta get users y la controlamos

  1. Creamos COntrolador Tenemos que crear el controlador: php artisan make:controller UserController

INFO Controller [app/Http/Controllers/UserController.php] created successfully.

y incluimos ese controlador en la api.php use App\Http\Controllers\UserController;

  1. Definimos función INDEX En el UserController Y tras ello volvemos a ejecutar el test
